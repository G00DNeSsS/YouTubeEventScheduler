## О проекте

**AutoDropVideo** — инструмент для YouTube-контентмейкеров, позволяющий заранее составить расписание публикаций и забыть об этом. Добавляете видео, назначаете время выхода, настраиваете интервалы — приложение работает в фоне и само загружает ролики на YouTube в нужный момент, даже если вы минимизировали окно или занялись другими делами.

Поддерживает несколько YouTube-аккаунтов, автоматически определяет формат Shorts, подставляет хэштег `#Shorts`, умеет работать с миниатюрами и следит за статусом каждой публикации в режиме реального времени.

---

## Возможности

- **Планирование публикаций** — выбираете дату и время старта, задаёте интервал (в минутах / часах / днях), получаете готовое расписание для нескольких видео сразу
- **Фоновая загрузка** — приложение работает в системном трее и автоматически загружает видео по расписанию без участия пользователя
- **Несколько аккаунтов** — поддержка нескольких YouTube-каналов с независимыми расписаниями
- **Автоопределение Shorts** — видео с вертикальным соотношением сторон и длительностью ≤ 60 сек автоматически помечаются как Shorts и получают хэштег
- **Управление метаданными** — название (с счётчиком символов), описание, теги, тип приватности (публичное / не в списке / приватное)
- **Миниатюры** — загрузка своей обложки или автоматическое извлечение кадра из видео через ffmpeg
- **Пакетная обработка** — добавление и редактирование сразу нескольких видео, шаблон нумерации в названии (`Влог {n}`)
- **Пропущенные публикации** — видео, которое не успело загрузиться по расписанию, автоматически публикуется при следующем запуске приложения
- **Календарь публикаций** — наглядный вид по датам с цветовой индикацией статусов
- **Очередь загрузок** — таблица с прогресс-барами, статусами и ссылками на опубликованные видео

## Описание модулей

### `api/youtube_uploader.py`
Загружает видео через YouTube Data API v3 с использованием возобновляемых (`resumable`) чанковых загрузок по 2 МБ. Собирает метаданные ролика (название, описание, теги, приватность, категория), автоматически добавляет `#Shorts` для коротких вертикальных видео и при необходимости устанавливает миниатюру. После загрузки возвращает URL видео.

### `auth/youtube_auth.py`
Реализует Google OAuth 2.0 Desktop Flow. Открывает браузер для авторизации, получает информацию о канале и статус верификации (разрешена ли загрузка видео длиннее 15 минут). Сериализует `Credentials` в JSON для хранения в SQLite и восстанавливает их при следующем запуске с автоматическим обновлением истёкшего `access_token`.

### `db/database.py` + `db/models.py`
Три таблицы:
- **`accounts`** — OAuth-токены, название канала, `channel_id`, статус верификации
- **`videos`** — путь к файлу, метаданные (название, описание, теги, приватность, тип), длительность, путь к миниатюре
- **`scheduled_posts`** — связь видео ↔ аккаунт ↔ дата/время, статус (`pending` / `uploading` / `done` / `failed`), `youtube_video_id`, ссылка

### `scheduler/task_scheduler.py`
Обёртка над APScheduler с `BackgroundScheduler` (UTC). При запуске загружает все `pending`-посты из базы: просроченные планируются с задержкой 10 сек (с интервалом 30 сек между ними), будущие — на точное время. По завершении загрузки обновляет статус и уведомляет UI через callback.

### `ui/main_window.py`
Главное окно с боковой панелью навигации и `QStackedWidget` для четырёх экранов. Управляет системным треем: при закрытии окна приложение сворачивается в трей и продолжает работу. При изменении статуса загрузки выводит уведомление в трее. Сигнал `post_status_changed` обновляет UI всех виджетов в реальном времени.

### `ui/video_editor.py`
Самый объёмный UI-модуль. Содержит:
- **`VideoLibraryWidget`** — список видео с поиском, сортировкой и мультивыбором
- **`VideoEditorDialog`** — добавление/редактирование видео с автоопределением типа через ffprobe
- **`SchedulePickerDialog`** — планировщик: выбор видео, аккаунта, даты старта и интервала, предпросмотр расписания
- **`BatchAddDialog`** — пакетное добавление файлов
- **`BulkEditDialog`** — массовое редактирование метаданных с шаблоном нумерации `{n}`

### `ui/calendar_widget.py`
Календарь на базе `QCalendarWidget` с цветными маркерами на датах. При клике на дату показывает список запланированных публикаций. Позволяет перенести публикацию на другое время или другой аккаунт.

### `ui/upload_queue.py`
Таблица всех постов с колонками: видео, аккаунт, дата, статус, URL, прогресс. Кнопки «Загрузить сейчас» и «Повторить» для ручного управления. Ссылку на опубликованное видео можно открыть в браузере или скопировать через контекстное меню.

### `ui/account_manager.py`
Список подключённых каналов. Отображает статус верификации аккаунта (влияет на доступность загрузки длинных видео). Кнопка «Добавить аккаунт» запускает OAuth-поток в браузере; удаление аккаунта каскадно очищает все связанные посты.

## Быстрый старт

### Требования

- Python 3.10+
- Windows 10 / 11
- ffmpeg / ffprobe в PATH *(опционально — для извлечения кадров и определения длительности)*
- Проект в [Google Cloud Console](https://console.cloud.google.com/) с включённым **YouTube Data API v3** и скачанным `client_secrets.json` (тип: Desktop App)

### Установка

```bash
git clone https://github.com/G00DNeSsS/AutoDropVideo.git
cd AutoDropVideo

python -m venv venv
venv\Scripts\activate

pip install -r requirements.txt
```

Поместите `client_secrets.json` в папку `config/`.

> **Важно:** файл `client_secrets.json` содержит секретные ключи OAuth — не публикуйте его в репозиторий.

### Запуск

```bash
python main.py
```

При первом добавлении аккаунта откроется браузер для авторизации через Google.

### Сборка .exe (Windows)

```bash
build.bat
```

Готовый исполняемый файл появится в папке `dist/`.

## Настройка Google API

1. Откройте [console.cloud.google.com](https://console.cloud.google.com/) и создайте новый проект
2. Перейдите в **APIs & Services → Library** и включите **YouTube Data API v3**
3. В **APIs & Services → Credentials** создайте **OAuth client ID** с типом **Desktop app**
4. Скачайте JSON, переименуйте в `client_secrets.json` и поместите в `config/`

## Рабочий процесс

```
1. Добавить аккаунт (OAuth)    →   Авторизация в браузере Google
2. Добавить видео              →   Метаданные, тип, миниатюра
3. Запланировать               →   Дата старта + интервал между роликами
4. Готово                      →   Приложение в трее публикует видео само
```
